Week 4 Blog Post

In which I discuss arrays and hashes in Ruby

February 28, 2016

Introduction

Welcome to the third post on my Dev Bootcamp blog! We've previously covered git/Github and the Box Model in CSS, and now we'll be moving on to some concepts in Ruby.

This week, I'm going to explore the differences between arrays and hashes in Ruby. These two Ruby objects are incredibly versatile and can help you a ton when writing out a program. Let's dive in...

Arrays

Whereas variables in Ruby point to a specific value, arrays point to a list, or collection, of values. An array is still considered an object like a variable, but it can hold more data and, therefore, do some more powerful things.

The array syntax is very simple. Let's take a look below at an array called my_array:

my_array = [1,2,3]

OK great! We've got an array. But wait, you must be wondering, what exactly is going on here? Well, let's break it down. As I mentioned above, I created a new array called my_array and assigned it, using the = sign, to a list of values. I chose the values 1, 2 and 3 as arbitrary values. An array can hold more or less than 3 values, it can even hold no values at all! As you can see, just like we assign a variable to a value, we use the same type of syntax to assign an array to a list of values, the only major difference is wrapping the list of values with brackets.

What's really cool about arrays is that, if we want, we could access an individual value within the array by simply calling its index. Whenver an array is created, it also comes along with index values, each of which points to a specific location within the array. Unlike in the real world, where we start counting from 1, in the world of Ruby, we start counting from zero. So, the first value in an array has an index number of 0. The second value has an index number of 1 and so on and so forth. As I mentioned above, an array can be set to an empty list, and in that case, if you called the 0 index, you will return the value nil.

How would one go about calling an index in an array. Well, it's pretty simple. Let's take a look at the syntax below:

my_array[1] => 2

So, I used our array from above my_array and called the index 1, returing the value of 2 from our example above. This is almost like passing an argument through a method; in this case, we are passing the index one to the array and it will spit back the value associated with that index. This can be incredibly helpful when trying to locate a particular value or when iterating over an array and you only want to execute code for a certain portion of the array (which can be denoted using the index number).

Hashes

The good thing is that hashes are incredibly similar to arrays in that they are an object that points to a list of values. The bad thing is that the syntax for hashes is completely different than for an array, however, we will see that maybe this way of storing values might be more beneficial for some complex programs.

Let's take a look at the hash syntax and then break it down. Here's our newly minted hash, my_hash:

my_hash = { "Hello" => "Goodbye" "First" => "Last" "Big" => "Small" }

Border

Next comes the border, which surrounds the padding layer and is used for basically stylistic reasons. The border layer, unlike the padding and margin layer, takes 3 different parameters: thickness, type and color. Thickness is very similar to setting the size of the padding, which can be adjusted in pixels or em. The higher the pixel or em number, the thicker, or bolder, the border will appear. In order for the border to appear, though, you need to set a type of border. The most common type of border is solid, which will display a solid line all around the padding of the content. You can also more stylized border types, like dashed, to create a unique look. And finally, if you don't want the default black border, you can adjust the color by inputting the specific nane, hex value, or rgb values to get the desired look. When writing this all our in CSS, you utilize the border selector and write out the three parameters as follows: thickness type color. No commas, no semi-colons, just spaces between each parameter.

Just like with padding, the border can be set all around the element by simply using the border selector. However, if you want to fine tune the border for a particular side, or just want a border on one side of the element, you can use the -top, -bottom, -left and -right suffixes to accomplish that.

Margin

Finally, we have the margin. This surrounds both the border and padding, with which it shares some similarities. While padding sets the white space around the content itself, the margin sets the white space around the element of the content. So, for this paragraph I'm writing, I am using a "p" tag HTML, so if I adjust the padding, that would effect the immediate white space around the text itself, and if I were to adjust the margin, that would effect the space around the "p" tag as a whole. The different between the text and the "p" tag is a bit ambiguous, but it makes a different within the HTML/CSS, particularly in relation to other elements on the page.

Since margin is the white space around an element, it is very useful for spacing and positioning, just like padding. Adjusting the margin of an element can help move an element in relation to it's nested parent or it's sibling within a nested parent. Just like padding, margin can be set on all sides of an element by using the maring selector, or can be fine tuned for each side using the suffixes mentioned above. Margin is a very powerful tool that can easily create space between elements or push over an element to fine tune positioning and help create the webpage layout you desire.

Conclusion

Alright, I think we learned a lot here. We learned about the box model, which describes that each HTML contains a series of layered boxes, margin, border and padding, each of which can be manipulated to create space, add style and positon elements. Utilizing these three parameters of the box model can take your CSS skills to the next level and make your websites that much more interesting.
