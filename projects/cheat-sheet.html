A quick reference for JavaScript objects and prototypes

JavaScript Prototypes

Basic idea: Protoypes are like the Ruby class in JavaScript; they provide the characteristics and basic ground work for any instance object created from that prototype

Two interrelated concepts: (i) protoype property and (ii) prototype attribute

(i) Prototype property: allows you to add properties/methods to an object

  • Syntax: object.prototype.name = value
  • object is the name of the object to add the property/method
  • prototype is standard syntax that refers to prototype for the object
  • name is the name of the new property/method
  • the value can be a function or other value (like an integer or string or array)

(ii) Prototype attribute: the characterisitc of the object; normally referred to as the prototype object, from which an instance object inherits its properties/methods

  • a prototype object is usually created as a constructor function, with various properties and attributes that can be passed down to instances of that prototype object
    • for example, the Array prototype object has certain properties/methods that are available to any new array object created
  • Constructor function syntax: function newConstructorName () {}
  • New object syntax: var newObject = new ConstructorName()
  • a constructor can take in arguments, which would be initialized when a new instance of the prototype object is created, and it can include properties/methods, each must be written as follows:
    • Property: this.propertyName = value
    • Function: this.functionName = function (){}

All inheritence is made possible through the prototype property

  • A prototype object will pass down any properties/methods as part of that constructor function
  • It will also pass down any properties/methods added using the prototype property syntax

We can access properties/functions with the prototype attribute

  • when accessing a property from an object (object.property) JS will first look to that object for the property
  • if JS does not find that property in that object, it will go up the chain to that object's prototype to find the property there
  • EX: if we create an object literal (var newObj = {}) and we call a property that doesn't currently exist in our created object, JS will go up the chain to that object's prototype object
    • in our case, newObj has Object as its prototype object, so JS will look there for any properties that are called that do not exist currently in the object itself

JavaScript Objects

Basic idea: Objects contain property/value pairs that can point to a number of JavaScript objects, such as strings, arrays, and functions

Two types of objects: (i) object literal and (ii) constructor functions

(i) Object literal: a way of creating an object that has property/value pairs; its prototype object is Object

  • Syntax: var newObj = { property1: value1, property2: value2 }

(ii) Constructor function: an object that is constructed as a function with property/value pairs; it can be a prototype object of its own

  • Syntax: function newConstructor = { this.property1 = value1; this.property2 = value2 }

Properties within objects can be assigned to objects themselves, as well as functions

Two types of call notation: (i) dot notation and (ii) bracket notation (both apply to object literals and constructor functions

(i) Dot notation: causes a literal call to occur where JS looks for the exact property name that is called on the object

  • Syntax: objectName.propertyName
  • if JS does not find the exact property name in the object, an error will be returned

(ii) Bracket notation: JS will evaluate the property name called and return the appropriate value based on that evaluation

  • Syntax: objectName["propertyName"]
  • JS will either find the exact name in the object or, if the given property name points to another value, JS will move down the chain of values and see if that property name is in the object
  • EX: if we have a variable var1 that is set equal to property1 of our newObj, then if we call newObj[var1], JS will see that var1 points to property1, which is in newObj and return the value
